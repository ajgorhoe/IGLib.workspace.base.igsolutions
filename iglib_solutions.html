1        Solutions based on IGLib

This document explains how solutions are arranged.

 

1.1    Various tasks performed on code

 

1.1.1.1  Refactoring

In order to refactor something in base libraries:

·         Open solution IGAll.sln (Section ‎1.2.1)

·         Perform refactoring

·         Compile solution (save all)

·         Copy changes in libraries from copies located in .../workspace/base/shelldev/lib to originals located in .../workspace/develop/shell/lib (Section ‎1.1.1.2)

·         Copy changes in application projects from copies located in .../workspace/base/shelldev/ extensions_shell to originals located in .../workspace/develop/shell and .../workspace/develop/shell/extensions (Section ‎1.1.1.2.1)

 

 

1.1.1.2  Updating changes in ANN shell libraries

In order to update changes in copies of ANN libraries in .../workspace/base/shelldev/lib such that they are reflected in originals in .../workspace/develop/shell/lib, do the following:

·         Open both projects (original from .../workspace/develop/shell/lib and a copy from .../workspace/base/shelldev/lib) from a location containing an up-to-date versions form SVN and that does not contain compiled code (only pure source, e.g. backup on Dropbox)

·         With Total commander, compare changes (the best is by content). If there is a risk of overriding something, compare individual files by contents.

·         Copy changed files from project copies (in shelldev directory) to project originals (in the shell directory).

·         Commit changes on SVN

 

In order to automatically update all changes (if you are sure that nothing vital is overridden):

·         Just copy all from .../workspace/develop/shell/lib to .../workspace/base/shelldev/lib

·         Commit changes to SVN

 

In order to update channges form original in .../workspace/develop/shell/lib to copy in .../workspace/base/shelldev/lib:

·         Follow a reverse procedure to that where source and target are replaced, just do it with more caution not to ovverride anything in the copy in .../workspace/base/shelldev/lib where things are much more complex and is subject to more maintenance effort.

 

1.1.1.2.1      Updating changes in ANN shell applications

Thinngs are very similar as in Section containing this one (i.e. in Section ‎1.1.1.2), except that locations are differennt. Namely:

In order to update changes in copies of ANN libraries in .../workspace/base/shelldev, you must copy projects from the following directories:

·         From .../workspace/base/shelldev/extensions_shell to .../workspace/develop/shell/extensions

·         From .../workspace/base/shelldev/extensions_shell/shell to .../workspace/develop/shell

 

1.1.1.3  Refactoring on a special version of ANN shell applications & libraries

 

In order to create a special refactored version of ANN shell applications, do the following:

·         Create a copy of .../workspace, e.g. .../testworkspace

·         Remove the .svn directories from the copy (such that canges can not be unintentionally committed)

·         Do fresh checkout (or copy an up to date local version without compiled files) of at least the following directories:

o       .../testworkspace/bin

o       .../testworkspace/develop/shell

o       .../testworkspace/base/shelldev

o       .../testworkspace/base/iglib

·         Open the solution .../testworkspace/base/shelldev/ ShellDevAll.sln

·         Do all refactoring in this solution

·         Rebuild the solution

o       WARNING: not everything will compile successfully! (projects form .../testworkspace/develop/shell may not compile because refactoring of base libraries will not affect them).

o       To fix this, you must update changes (see the next main bullet) and re/compile!

·         Update changes form all copies of projects from .../testworkspace/base/shelldev to the corresponding original locations in .../testworkspace/ (such as in Sections ‎1.1.1.2‎ and 1.1.1.2.1, but within the copy of the workspace directory, i.e. within the .../testworkspace/)

o       Since this task is performed on a temporary copy of the workspace directory, copying can be performed directly without verification of which changes have been made and whether something could be overridden unintentionally.

·         Open the solution .../workspace/develop/shell/extended_shell.sln

·         Rebuild this solution perform further tasks according to what you want to achieve

o       If you want to create refactored applications or libraries, run the appropriate creation script.
